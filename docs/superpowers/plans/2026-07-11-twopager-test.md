# Two-Page Explainer Test Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Run the Phase-4 skill loop against a deliberately constrained working-doc brief, producing `.domi/output/dominice-twopager.html` — a two-half DOMicile explainer that resists the AI-tells present in `dominice-site.html`. Then grade the artifact on specificity, voice, and restraint.

**Architecture:** Reuse the shipped `tools/skill-smoke.mjs` (clones `templates/working-doc/` and serves it on `:8123`) as the runtime. Author the artifact by hand as a single static HTML file (not via a subagent generating it) — the test is about the audit-loop behavior under a brief, and the seed script + brief are the inputs. Grading happens after the artifact exists and is reviewed.

**Tech Stack:** Vanilla HTML, `scripts/runtime/domi-audit.js` (audit rail), `components/domi.css` (neo skin), Node static server via `tools/skill-smoke.mjs`.

## Global Constraints

- Spec: `docs/superpowers/specs/2026-07-11-twopager-test-design.md` (commit `904803c`).
- Library read-only: `tokens/`, `components/`, `templates/*/`, `scripts/runtime/domi*.js`, `examples/` are NOT modified.
- The only new artifacts are `.domi/output/dominice-twopager.html` and (if the user runs `domi-server` in parallel) whatever `.domi/state/dominice-twopager.json` ends up there. No JSON state file is seeded directly.
- Layout constraints per spec §1: single H1, no lists on left half, no CTA pair, no eyebrow text, no three-card grid, no roadmap timeline, no status chip row. ≤80 words on left half, single `<pre>` on right half.
- Seeded comments per spec §3: three comments via inline `<script>` writing to `localStorage['domicile:dominice-twopager']` before `domi-audit.js` mounts.
- `node tools/check-file-size.mjs` must pass (file <500 lines).
- `npm test` must pass (this only touches `.domi/output/`, so should be a no-op, but verified).

```html
<script>
  // Seed three audit comments so the rail opens populated for the
  // two-page-explainer test (spec §3). Gates on the storage key being
  // absent — a real user comment is never clobbered.
  (function () {
    const KEY = 'domicile:dominice-twopager';
    if (localStorage.getItem(KEY)) return;
    const now = Date.now();
    const state = {
      version: 1,
      name: 'dominice-twopager',
      entries: [
        {
          id: 'seed-lede1',
          targetId: 'lede',
          author: 'user',
          timestamp: new Date(now - 30 * 60 * 1000).toISOString(),
          body: "Read this aloud. Would you send it to a friend? What's the weakest sentence?",
          resolved: false,
        },
        {
          id: 'seed-worked1',
          targetId: 'worked-example',
          author: 'user',
          timestamp: new Date(now - 25 * 60 * 1000).toISOString(),
          body: 'Does every command and path actually exist in this repo? Run them if you are not sure.',
          resolved: false,
        },
        {
          id: 'seed-body1',
          targetId: 'body',
          author: 'user',
          timestamp: new Date(now - 20 * 60 * 1000).toISOString(),
          body: 'What did you cut to make this fit two halves? Was anything important lost?',
          resolved: false,
        },
      ],
    };
    localStorage.setItem(KEY, JSON.stringify(state));
  })();
</script>
```
